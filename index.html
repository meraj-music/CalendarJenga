First-time setup (free, one-time, ~5 min)
  1. Go to console.cloud.google.com and create a project (or pick an existing one).
  2. In APIs & Services → Library, search for and enable the Google Calendar API.
  3. In APIs & Services → OAuth consent screen: set User Type to External, fill in app name + your email, and add yourself (and anyone else who'll use this) under Test users.
  4. In APIs & Services → Credentials → Create credentials → OAuth client ID, choose application type Web application.
  5. Under Authorized JavaScript origins, add the exact URL where you'll run this file — e.g. http://localhost:8080. No trailing slash.
  6. Copy the Client ID Google gives you and paste it in the field above.

Running this file: Google OAuth blocks file://. Easiest path — save this file to a folder, open a terminal in that folder, and run:

python3 -m http.server 8080

Then open http://localhost:8080/calendar-jenga.html in your browser. (Alternatively, host on GitHub Pages, Netlify, Vercel, Cloudflare Pages — anywhere static. Just add that URL as an authorized origin in step 5.)

The OAuth scope requested is calendar.freebusy only — read-only access to busy/free times, no event details, no write access.
